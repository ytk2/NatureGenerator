# v0.5.0 Product Baseline

## Release Identity

- **Git tag:** `v0.5.0`
- **Release name:** Interactive Generation Command
- **Status:** Current stable baseline
- **Tested platform:** macOS
- **Tested host:** Autodesk Fusion

The immutable `v0.5.0` Git tag identifies the exact released source. This
document lives on the current development branch and describes how to verify,
recover, and build from that stable product baseline; it does not change the
tagged commit.

The resolution-41 acceptance run observed 44,396 vertices, 86,400 faces, and an
elapsed time of approximately 1.869 seconds in Autodesk Fusion on macOS.

## Key Architecture Boundaries

- `core/` owns dependency-free voxel sampling, mesh extraction, mesh processing,
  validation, statistics, and exporters.
- `presets/` contains immutable user-facing metadata and stable `generator_id`
  strings; presets do not import generator implementations.
- `generators/` resolves a Fusion-independent `GenerationRequest` and produces a
  `TriangleMesh` and `GeneratorResult` without importing Autodesk APIs.
- `commands/` contains Fusion-independent generation orchestration.
- `fusion/` owns functional `adsk` integration, Fusion command lifecycle, and
  `TriangleMesh`-to-`MeshBody` conversion.
- `NatureGenerator.py` contains only the entry-point loader bootstrap and fatal
  Fusion lifecycle diagnostics required to start the add-in reliably.

See [ARCHITECTURE.md](../ARCHITECTURE.md) for the complete dependency model.

## Known Limitations

- Sponge is the only executable preset.
- There is no live preview or progress indicator.
- Generation is synchronous and pure Python.
- The command remains under Utilities > Add-Ins.
- Finite gyroid meshes have open crop boundaries.
- There is no smoothing, decimation, or installation package.

## Recovering the Baseline

To inspect the exact released source:

```bash
git checkout v0.5.0
```

Checking out a tag creates a detached `HEAD`. It is appropriate for inspection,
testing, or recovery, but should not be used directly for new development or
commits.

Start new feature development from the latest `main` instead:

```bash
git checkout main
git pull origin main
git checkout -b feature/<name>
```
